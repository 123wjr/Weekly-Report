# HTTP 协议学习笔记总结

------

# 一、HTTP 协议概述

## 1️⃣ 什么是 HTTP？

**HTTP（HyperText Transfer Protocol）超文本传输协议**

作用：
 👉 浏览器 和 服务器 之间传输数据的规则

简单理解：

```
浏览器 ——(HTTP请求)——> 服务器
浏览器 <——(HTTP响应)—— 服务器
```

------

## 2️⃣ HTTP 特点（⭐常考）

| 特点          | 说明                             |
| ------------- | -------------------------------- |
| 无状态        | 每次请求互相独立，不记住之前操作 |
| 基于请求-响应 | 必须一问一答                     |
| 明文传输      | 不安全（HTTPS 才加密）           |
| 简单快速      | 报文结构简单                     |

------

## 3️⃣ HTTP 版本

| 版本       | 特点               |
| ---------- | ------------------ |
| HTTP/1.0   | 每次请求都新建连接 |
| HTTP/1.1 ⭐ | 长连接 keep-alive  |
| HTTP/2     | 多路复用、更快     |
| HTTP/3     | 基于 QUIC（UDP）   |

------

------

# 二、HTTP 请求协议（Request）

## 1️⃣ 请求报文结构（⭐必会）

```
请求行
请求头
空行
请求体
```

------

## 2️⃣ 请求行（3部分）

格式：

```
请求方法  请求路径  协议版本
```

示例：

```
GET /user/list HTTP/1.1
```

------

## 3️⃣ 请求方法（⭐重点）

| 方法    | 作用     |
| ------- | -------- |
| GET     | 查询数据 |
| POST    | 提交数据 |
| PUT     | 修改数据 |
| DELETE  | 删除数据 |
| PATCH   | 局部修改 |
| OPTIONS | 预检请求 |

口诀：
 👉 查GET，增POST，改PUT，删DELETE

------

## 4️⃣ 常见请求头

| 请求头         | 作用         |
| -------------- | ------------ |
| Host           | 服务器地址   |
| User-Agent     | 浏览器信息   |
| Cookie         | 携带会话信息 |
| Content-Type   | 数据类型     |
| Content-Length | 数据长度     |

示例：

```
Host: localhost:8080
Content-Type: application/json
```

------

## 5️⃣ 请求体（Body）

👉 只有 POST/PUT 等才有

常见格式：

```
application/json
application/x-www-form-urlencoded
multipart/form-data
```

------

------

# 三、HTTP 响应协议（Response）

## 1️⃣ 响应报文结构

```
状态行
响应头
空行
响应体
```

------

## 2️⃣ 状态行（3部分）

格式：

```
协议版本  状态码  描述
```

示例：

```
HTTP/1.1 200 OK
```

------

## 3️⃣ 状态码（⭐⭐⭐必背）

### 1xx 信息

- 100 Continue

### 2xx 成功

| 状态码 | 含义     |
| ------ | -------- |
| 200    | 成功     |
| 201    | 创建成功 |
| 204    | 无内容   |

------

### 3xx 重定向

| 状态码 | 含义       |
| ------ | ---------- |
| 301    | 永久重定向 |
| 302    | 临时重定向 |
| 304    | 缓存命中   |

------

### 4xx 客户端错误

| 状态码 | 含义       |
| ------ | ---------- |
| 400    | 请求错误   |
| 401    | 未认证     |
| 403    | 禁止访问   |
| 404 ⭐  | 资源不存在 |

------

### 5xx 服务器错误

| 状态码 | 含义       |
| ------ | ---------- |
| 500    | 服务器异常 |
| 502    | 网关错误   |
| 503    | 服务不可用 |

------

## 4️⃣ 常见响应头

| 响应头         | 作用       |
| -------------- | ---------- |
| Content-Type   | 数据格式   |
| Content-Length | 数据长度   |
| Set-Cookie     | 设置Cookie |
| Location       | 重定向地址 |

------

## 5️⃣ 响应体

👉 返回给浏览器的数据

可能是：

- HTML
- JSON ⭐（接口开发最常见）
- 图片
- 文件

------

# 四、HTTP 协议解析（开发必懂）

------

## 1️⃣ 一次完整流程（⭐面试常问）

```
① 浏览器输入URL
② DNS解析IP
③ 建立TCP连接（三次握手）
④ 发送HTTP请求
⑤ 服务器处理
⑥ 返回HTTP响应
⑦ 浏览器解析页面
```

------

## 2️⃣ GET vs POST 区别

| 对比     | GET  | POST     |
| -------- | ---- | -------- |
| 参数位置 | URL  | Body     |
| 安全性   | 低   | 相对高   |
| 长度限制 | 有   | 基本无限 |
| 是否缓存 | 会   | 不会     |
| 用途     | 查询 | 提交     |

------

## 3️⃣ Cookie 与 Session

### Cookie

- 存浏览器
- 体积小
- 不安全

### Session

- 存服务器
- 安全
- 需要 Cookie 传 sessionId

关系：

```
Cookie 里保存 sessionId
session 数据在服务器
```

------

## 4️⃣ HTTP vs HTTPS

| HTTP   | HTTPS   |
| ------ | ------- |
| 明文   | 加密    |
| 不安全 | 安全    |
| 80端口 | 443端口 |
| 无证书 | SSL证书 |

👉 实际开发必须用 HTTPS
